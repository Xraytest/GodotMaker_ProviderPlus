# CLAUDE.md

## Project

This is a Godot game project generated by GodotMaker.
To build or modify this game, invoke the `/orchestrator` skill.

## Architecture

- ECS: Components (pure data) + Systems (logic) via gecs
- Scene tree reserved for UI/menus; gameplay entities use ECS
- Scene markers for level design → converted to ECS entities at runtime

## Key Files

- `PLAN.md` — Task breakdown, risk analysis, progress tracking
- `STRUCTURE.md` — Architecture: Components, Systems, Entity Archetypes
- `ASSETS.md` — Art direction and asset inventory
- `SCENES.md` — Scene/UI layout descriptions with element positions
- `MEMORY.md` — Index linking to memory/ sub-files (discoveries, decisions, gotchas)
- `TOC.md` — Document index

## Conventions

- GDScript for all game logic; English for code and comments
- TDD: write tests alongside implementation
- One System per file (`{name}_system.gd`), one Component per file (`{name}.gd`)
- Tests in `test/`, named `test_{name}.gd`
- Physics callbacks must never manipulate the node tree directly — use deferred calls
- Only one System writes Transform per entity
- Scene tree is for UI/menus only; gameplay entities use ECS
- All systems must have corresponding unit tests
- Game must pass e2e tests before considered complete
