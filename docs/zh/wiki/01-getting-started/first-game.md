# 你的第一款游戏

这篇教程带你从一个空文件夹，一步步做出一个可以运行的 Godot 游戏。整个过程只需要在 Claude Code 里输入九条指令——每一步对应一条。指令发出去之后，AI 会在后台干活，你只需要等；它遇到问题问你，或者你想看看进度时，再介入就行。

**大概要花多久：** 一个小游戏（比如弹球、简单的横版跳跳游戏、基础解谜）自己实际花的时间大约 30 分钟，分散在整个流程里——AI 在两步之间跑的时间不算在内。游戏越复杂越久，但指令是一样的。

随时可以停下来，改天再回来继续。每条指令都会检查已经完成了什么，然后从断点继续。

## 开始前

先完成[安装](installation.md)。你需要：

- 五款工具都已安装，且 `python tools/check_env.py` 全部通过
- 环境变量里已设置 `GOOGLE_API_KEY`
- 已在本地克隆 GodotMaker 仓库

## 创建游戏项目文件夹

GodotMaker 不在它自己的仓库里运行。你需要新建一个空文件夹用来放你的游戏，然后在 GodotMaker 文件夹里运行一条命令，把 AI 需要的所有东西都安装到那个新文件夹里。

在 GodotMaker 仓库目录下执行：

```bash
python tools/publish.py /path/to/my-game
```

把 `/path/to/my-game` 替换成实际路径，比如 Windows 上的 `C:\Games\my-bouncing-ball`，或者 macOS/Linux 上的 `~/games/my-bouncing-ball`。文件夹不存在会自动创建。

这条命令会把 AI 技能、钩子脚本、配置文件和模板都复制到你的新游戏文件夹。如果它找不到 Godot，会问你 Godot 可执行文件的完整路径。

完成之后，在新游戏文件夹里打开 Claude Code：

```bash
cd /path/to/my-game
claude
```

从这里开始，所有操作都在这个 Claude Code 会话里进行。

---

## 第 1 步 — `/gm-scaffold`

**你输入的内容：**

```
/gm-scaffold
```

**会发生什么：** AI 创建空的 Godot 项目结构——文件夹、必要插件（gecs 负责游戏逻辑，gdUnit4 负责测试），以及第一个 Git 提交。这一步不会问你任何问题，自动完成。

**生成的文件：** `project.godot`、`addons/`、`src/`、`scenes/`、`assets/`、`tests/`、`e2e/conftest.py`。

**怎么知道完成了：** Claude Code 打印出汇总信息并回到提示符。文件夹里会出现一个新的 `project.godot` 文件。

每个项目只需要运行一次。如果在已有的项目上再次运行，它会检测到脚手架已存在，安全地跳过。

---

## 第 2 步 — `/gm-gdd`

**你输入的内容：**

```
/gm-gdd
```

**会发生什么：** AI 会和你聊聊你想做的游戏。它会问一些问题，比如：游戏目标是什么？玩家做什么操作？画风是什么感觉？有几关？你不需要提前准备好答案——说你知道的就行，剩下的 AI 会用合理的默认值填上。

访谈结束后，它会写出所有规划文档：游戏设计文档、任务清单、文件夹结构方案、场景列表和素材列表。

**生成的文件：** `GDD.md`、`PLAN.md`、`STRUCTURE.md`、`SCENES.md`、`ASSETS.md`、`TOC.md`。

**怎么知道完成了：** AI 会总结设计方案，问你满不满意。翻一翻 `GDD.md`——这是正式开始制作前纠正方向的好时机。输入你的反馈，或者说可以继续。

---

## 第 3 步 — `/gm-asset`

**你输入的内容：**

```
/gm-asset
```

**会发生什么：** AI 用 Google Gemini 生成游戏美术——精灵图、背景、图标。如果你想用自己的图，可以先把文件放到 `assets/` 文件夹里，AI 会分析已有的素材，只补全缺少的部分。

**生成的文件：** `assets/` 里的图片文件，以及更新后的 `ASSETS.md`。

**怎么知道完成了：** AI 会报告哪些素材是新生成的、哪些是已有的。你可以打开 `assets/` 文件夹看看图片效果；如果哪张不对，告诉 AI，它会重新生成那几张。

---

## 第 4 步 — `/gm-build`

**你输入的内容：**

```
/gm-build
```

**会发生什么：** 这是最长的一步。AI 通过把工作交给专项子智能体来实现游戏——可以把它们理解为助手，每个负责游戏的一个部分。每个任务都会自动经历三层处理：一个子智能体写代码，第二个验证代码能编译且测试通过，第三个检查常见的 Godot 坑。你不需要盯着它——等就行。

输出会很多，任务一条条发出去、一条条报回来，这很正常。

**生成的文件：** `src/` 里的游戏代码、`scenes/` 里的场景文件、`tests/` 里的单元测试。

**怎么知道完成了：** AI 打印出构建汇总。如果有任务没通过检查，会在汇总里注明，留给 `/gm-fixgap` 处理。

---

## 第 5 步 — `/gm-verify`

**你输入的内容：**

```
/gm-verify
```

**会发生什么：** AI 做一次机械式检查——项目能不能无报错地编译？单元测试通过了吗？有没有缺少必要文件？这是一个快速的自动化步骤，不会问你问题。

**生成的文件：** 全部检查通过后，向 `.godotmaker/stage.jsonl` 追加一个 `verify` 事件。详细报告打印在聊天里——磁盘上不会再写其他文件。

**怎么知道完成了：** AI 打印通过/失败汇总。如果这里有失败，说明构建步骤有东西需要修，接下来两条命令来处理。

---

## 第 6 步 — `/gm-evaluate`

**你输入的内容：**

```
/gm-evaluate
```

**会发生什么：** AI 在后台运行实际的游戏（不会在你屏幕上弹出窗口），截图，然后对照游戏设计文档里的描述给结果打分。这是一个独立视角，用来判断游戏的样子和行为是否正确——和创建它的构建步骤完全分开。

**生成的文件：** `.godotmaker/evaluation.json`，截图保存在 `e2e/screenshots/`。

**怎么知道完成了：** AI 打印评估分数，以及与设计不符的地方的清单。你可以打开截图看看 AI 看到的画面。

---

## 第 7 步 — `/gm-fixgap`

**你输入的内容：**

```
/gm-fixgap
```

**会发生什么：** AI 读取评估结果，判断需要改什么，然后派子智能体逐一修复——和构建步骤一样的三层流程。如果评估没有发现问题，这条指令可以跳过。

**生成的文件：** 更新后的游戏代码，以及记录修复内容的 `GAP.md`（完成后归档到 `.godotmaker/gaps/`）。

**怎么知道完成了：** AI 总结修了什么。如果改动比较大，可以再跑一次 `/gm-verify` 和 `/gm-evaluate` 确认修复到位。

---

## 第 8 步 — `/gm-accept`

**你输入的内容：**

```
/gm-accept
```

**会发生什么：** AI 展示游戏当前的状态，让你做个决定：

- **接受** — 这个 tag 满意了，继续收尾。
- **拒绝（回退循环）** — 还有问题，AI 会回到 `/gm-fixgap` 再试一次。
- **暂停** — 想在这里结束本次会话，以后再回来。

**生成的文件：** `.godotmaker/stage.jsonl` 里的一条验收记录。

**怎么知道完成了：** 你做出了选择并告诉了 AI。

---

## 第 9 步 — `/gm-finalize`

**你输入的内容：**

```
/gm-finalize
```

**会发生什么：** AI 做收尾工作——把当前 tag 的工作文档归档到 `docs/tags/<Tag>/`、写一份按 tag 的 changelog、在本地执行 `git tag <Tag>`，并重置每 tag 的运行时状态以便下一个 `/gm-gdd` 在干净状态下开始。这一步很短，自动完成。

**生成的文件：** `docs/tags/<Tag>/` 归档、`.godotmaker/final_report.json`、本地 git tag。

**怎么知道完成了：** AI 确认 tag 已封口，项目可以在 Godot 里打开了。

---

## 你现在有一个可以运行的游戏了

在 Godot 编辑器里打开项目：

```bash
godot --editor --path /path/to/my-game
```

或者直接运行：

```bash
godot --path /path/to/my-game
```

以后想加新功能，在同一个文件夹里开一个新的 Claude Code 会话，从 `/gm-gdd` 重新开始就行。AI 会先读 `MEMORY.md`，了解已经做了什么，再规划下一步改动。`/gm-scaffold` 只需要做一次，不需要重复。

想了解项目文件夹里每个东西的含义，见[项目结构](project-layout.md)。
